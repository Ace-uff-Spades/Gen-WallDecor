How should we organize the Figma wireframes?

7 frames total: Landing · Create ×3 · Generate · Wall View · History

A

Linear Canvas — All frames in one row

All 7 screens laid out left-to-right on a single Figma page in flow order. Simple to navigate, easy to screenshot the whole flow at once.

Landing
→
Create 1
→

→
Create 3
→

→

→
History

Best for: quick overview, sharing a single screenshot

B

Sectioned Canvas — Grouped by feature area

Frames grouped into labeled sections on one page: Onboarding (Landing), Creation Flow (Create 1–3, Generate), Result (Wall View), Discovery (History). Good for annotating redesign areas by feature.

Onboarding
Landing
Creation Flow
Create 1–3 · Generate
Result
Wall View
Discovery
History

Best for: redesign planning, annotating per feature area

C

Multi-page Figma File — One Figma page per screen

Each of the 7 screens gets its own Figma page. Max focus per screen, easy to add lots of annotations without clutter. Harder to see the big picture.

📄 Landing
📄 Create – Step 1
📄 Create – Step 2
📄 ...

Best for: deep per-screen redesign work

My recommendation: B (Sectioned Canvas) — grouping by feature area makes it easy to annotate redesign ideas per section without losing the flow context.
